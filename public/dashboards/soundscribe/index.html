目前使用者工作流程

01

輸入

檔案、URL、live dictation、watch folder rule，或 Shortcut/App Intent。

02

準備音訊

驗證 media、將 URL download streaming 到檔案，並以 AVFoundation-native path 轉換。

03

分段

VAD、30-90 秒 chunk size、overlap，以及基於 sample range 的長音訊 loop。

04

推理

在 Apple Silicon 上透過 qwen3-asr-swift 與 MLX/Metal 執行 Qwen3-ASR。

05

後處理

Confidence filter、BPE mojibake repair、repetition collapse、標點與 deduplication。

06

保存

Canonical txt transcript、可選 subtitle 與 speaker sidecars、history library entry、exports。

核心設計原則

原則目前落地方式
App Store-safe automation根據 AGENTS.md，只做 Shortcuts 與 watch folders；不加入 CLI helper、system audio、screen recording 或 network server。
Canonical output主 transcript 永遠是 *_transcript.txt；subtitle 與 speaker files 是 sidecars。
On-device firstASR inference 透過 MLX 在 Apple Silicon 本地執行；AI post-processing 是可選且明確 provider-specific。
可恢復 workflowQueue persistence v2、history library、unique output paths，以及 file-backed records。

模組細節

轉錄與音訊路徑

NativeTranscriptionService.swift 負責載入模型並協調 chunked native ASR。LongAudioTranscriber.swift 處理 mel extraction 與 GPU inference。AudioConverter.swift 與 AudioStreamManager.swift 覆蓋 file conversion 與 live dictation audio。

Queue、持久化與輸出

TranscriptionManager.swift 掌管 batch state、crash recovery 與 background diarization。OutputPathResolver.swift 避免檔名碰撞。BatchTypes.swift 承載 subtitle、speaker metadata 與 source-kind state。

History、watch folders 與 Shortcuts

TranscriptLibraryStore.swift 寫入 history JSON library。WatchFolderService.swift 提供 non-recursive folder monitoring 與 polling fallback。AppShortcuts.swift 暴露 Transcribe File、Transcribe URL 與 recent transcript intents。

AI、export 與 speaker tools

GoogleAIService.swift、AIModelFetcher.swift 與 AITypes.swift 負責 route selected AI providers。ExportService.swift 與 ExportPresetStore.swift 處理 TXT/PDF/Markdown/SRT/VTT/JSON exports。SpeakerEditorSheet.swift 與 SpeakerMetadata.swift 處理 rename/merge metadata。
